Noo Noo Adventure - Game Engine

PROJECT TYPE

University Project - C++, GLFW (OpenGL), LUA, ImGUI, Stb_image, CMAKE, GitHub

Description

A small demo game, "Noo Noo's Adventure", used to demonstrate the game engine we developed throughout the semester. The game follows Noo Noo the vacuum as you try and survive against the evil teletubbies.

Features Developed by Me

Terrain

Implemented heightmap and fault noise generation to create realistic and dynamic terrain. Designed the texturing logic to allow the terrain to be textured with four combined textures based on varying heights, enhancing visual diversity and realism in the environment.

GUI

Developed a facade for the IMGUI library, enabling users to add GUI elements via LUA scripts. This system supports various interactive components, such as buttons, allowing for flexible and dynamic user interfaces within the game.

Water

Engineered the water rendering and creation logic, incorporating advanced shader logic with DUDV maps, frame buffers, and reflection and refraction textures. This results in realistic and visually appealing water effects that enhance the game's immersive experience.

Graphics Library Abstraction

Created a facade for the GLFW/OpenGL library, allowing for the use of multiple window types. This abstraction ensures the engine's flexibility and adaptability for future enhancements and changes.

Skybox

Managed the skybox system, handling all aspects of texturing and rendering. This feature creates a seamless and immersive background environment, contributing to the overall aesthetic and depth of the game world.

Camera System

Developed the complete camera system, writing the class and incorporating all necessary logic for smooth and dynamic camera movements. This system supports various camera behaviors, enhancing the player's visual experience and interaction within the game.

Rendering Logic

Contributed to the mesh rendering process by iterating through all entities with meshes and implementing the logic to render them. Developed the texture manager system for efficient loading and usage of textures, the shader manager for handling shader programs, and the material manager for managing material properties and components. These systems collectively ensure efficient and high-quality rendering within the game engine.
